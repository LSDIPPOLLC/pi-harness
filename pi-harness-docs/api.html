API Reference

Configuration schema, settings structure, and hook interfaces.

Settings Schema

settings.json Root Configuration
▼

Main configuration file for pi harness. Located at .pi/settings.json.

.pi/settings.json 
{
  "skills": [string],
  "prompts": [string],
  "permissions": {
    "allow": [string]
  },
  "hooks": [{
    "type": "PostToolUse"|"PreToolUse"|"Stop",
    "matcher": string,
    "command": string
  }]
}
permissions Allow Patterns
▼

Permission patterns for auto-allowed operations.

PatternDescription
Bash(npm test *)Allow npm test with any args
Bash(git status *)Allow git status with any args
ReadAllow all file reads
EditAllow all file edits
WriteAllow all file writes

Hooks Interface

Hook Types PreToolUse, PostToolUse, Stop
▼

Three hook types fire at different points in pi's execution.

TypeWhenUse Case
PreToolUseBefore tool executesBlock dangerous ops
PostToolUseAfter tool executesFormat files, run linters
StopConversation endsHealth checks, drift detection
Exit Codes Hook Behavior
▼

Hook scripts return codes that control pi's behavior.

CodeMeaningEffect
0SuccessOperation proceeds normally
2BlockOperation blocked, stdout shown
otherErrorTreated as failure, operation proceeds
Environment Hook Variables
▼

Environment variables passed to hook scripts.

VariableDescription
CLAUDE_TOOL_INPUTJSON string of tool input parameters
CLAUDE_TOOL_OUTPUTJSON string of tool output (PostToolUse only)
CLAUDE_PROJECT_DIRProject root directory

Prompts Interface

prompts/*.md Prompt Format
▼

Prompt templates in .pi/prompts/ loaded via /prompt-name.

.pi/prompts/project.md 
---
description: Run the test suite
---
Run tests and report pass/fail counts.

Usage: `/test` for full suite, `/test <pattern>` for specific.
Arguments Prompt Variables
▼

Templates support positional arguments.

VariableDescription
$1, $2, ...Positional arguments
$@ or $ARGUMENTSAll args joined
${@:N}Args from Nth position
${@:N:L}L args starting at N
