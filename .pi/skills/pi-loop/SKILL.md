# Pi Loop

Continuous improvement cycle for harness maintenance.

## Loop Steps

1. **Audit** — Score current state via `/skill:pi-audit`
2. **Identify** — Find weakest pillar
3. **Improve** — Apply targeted fixes
4. **Validate** — Verify improvements
5. **Repeat** — Next iteration

## When to Run

- Weekly maintenance
- After major project changes
- When issues arise
- During onboarding to new projects

## Focus Areas

Rotate focus across pillars:
- Week 1: Context Engineering
- Week 2: Quality Gates
- Week 3: Permissions
- Week 4: Ergonomics
