# Pi Memory

Design cross-session memory for persistent knowledge.

## Memory Types

### User Memory
- Role and expertise level
- Interaction preferences
- Communication style

### Feedback Memory
- What worked (confirmations)
- What didn't (corrections)
- Project-specific patterns

### Project Memory
- Ongoing work status
- Key decisions
- Deadlines and constraints

### Reference Memory
- External systems
- URLs and documentation
- Configuration details

## Memory Index

`.pi/memory/MEMORY.md` indexes all memories:

```markdown
# Memory Index

## User
- [Role](user_role.md)

## Feedback
- [Testing](feedback_testing.md)

## Project
- [Context](project_context.md)
```

## Maintenance

- Review monthly
- Remove stale entries
- Update as project evolves
- Archive completed items
